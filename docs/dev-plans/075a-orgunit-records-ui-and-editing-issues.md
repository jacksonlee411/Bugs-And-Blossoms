# DEV-PLAN-075A：OrgUnit 记录新增/插入 UI 与可编辑字段问题记录

**状态**: 规划中（2026-02-06 00:37 UTC）

## 背景与问题
- 当前 OrgUnit 详情页的“新增记录”功能只允许修改名称，未覆盖所有应支持的可修改字段。
- “插入记录”和“修正记录”入口在同一页面区域混杂，造成操作语义不清晰。

## 目标
- 明确问题范围，形成可执行的修复方向与验收口径。
- 为后续契约与 UI 调整提供记录入口，避免口头需求漂移。

## 问题清单
1. **新增记录可编辑字段不足**
   - 现状：仅允许修改名称。
   - 期望：新增记录应支持页面中“可修改字段”的完整集合（与后端允许字段一致）。
   - 影响：用户无法在新增记录时一次性补齐必要信息，导致多次修正或手工绕行。

2. **插入记录与修正记录 UI 混杂**
   - 现状：插入记录与修正记录在同一区域/流程混用，难以区分“新版本插入”与“纠错修正”。
   - 影响：用户容易误操作，且与事件语义（新增 vs 更正）不一致。

## 评估方向（待确认）
- **字段覆盖**：梳理“可修改字段”与 UI 表单映射，确保新增记录与修正记录的字段集一致且受后端校验约束。
- **交互拆分**：将“插入记录”与“修正记录”拆分为不同区域或不同步骤，明确操作语义与提示文案。

## 实施步骤（待确认）
1. [ ] 盘点后端允许字段（新增/插入/修正）与当前 UI 字段差异。
2. [ ] 明确 UI 交互方案（分区/分步/按钮语义）。
3. [ ] 更新契约与文档（引用 SSOT，不复制门禁矩阵）。
4. [ ] 实现 UI 与后端对齐，并补充测试。
5. [ ] 记录验收与执行日志。
