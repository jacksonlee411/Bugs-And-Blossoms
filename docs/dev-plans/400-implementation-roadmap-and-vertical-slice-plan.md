# DEV-PLAN-400：实施路线图与垂直切片编排计划

**状态**: 规划中（2026-03-17 07:23 CST）

## 1. 背景与上下文

随着 `300/310/320/330/340/350/360/370/380/390` 逐步形成，一定需要一个统一的“怎么开始做、先做什么、哪些并行、如何验收”的总编排计划。

`400` 的职责不是重复业务设计，而是回答：

- 什么先做
- 什么后做
- 哪些能力可以并行
- 哪些能力必须形成垂直切片后才算完成

## 2. 目标与非目标

### 2.1 核心目标

- [ ] 建立从平台基座到业务交付的统一实施顺序。
- [ ] 以垂直切片而非模块堆叠作为主要交付节奏。
- [ ] 明确各阶段的 MVP、阻塞项与验收标准。

### 2.2 非目标

- [ ] 本计划不替代各子计划的详细设计。
- [ ] 本计划不定义具体 schema、API 或页面细节。

## 5. 关键设计决策

### 5.1 垂直切片优先（选定）

- 任何阶段都要形成用户可见、可操作的最小闭环。
- 不接受“后端先堆完再补 UI”的长期分叉。

### 5.2 横切计划前置但不过度阻塞（选定）

- `310/320/330` 应尽早冻结原则。
- 但实现深度应与阶段目标匹配，不做超前工程化。

## 6. 验收标准

- [ ] 各子计划有清晰的前后依赖与并行关系。
- [ ] 系统交付以真实用户切片为主，而不是仅以模块完成度统计。
- [ ] 项目启动后能据此继续拆出阶段性交付计划。
